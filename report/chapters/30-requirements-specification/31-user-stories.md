# Capítulo III: Requirements Specification

## User Stories

En esta sección se definen los requisitos del ecosistema mediante Épicas e Historias de Usuario, estructuradas a partir del Big Picture generado en nuestra sesión de Event Storming. Los requerimientos abarcan las funcionalidades del dominio transaccional, el sitio web estático (Landing Page) enfocado en la conversión de roles segmentados, y las Technical Stories. Todos los criterios de aceptación son comprobables y siguen estrictamente la estructura BDD (Gherkin: Given-When-Then) en tiempo presente y tercera persona, incluyendo los escenarios de interacción request/response para el backend.


| **Epic ID** | **Título**                                                              | **Descripción**                                                                                                                                                                                                                                                                                                       |
| ----------- | ----------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **EP01**    | Gestión de identidad y onboarding personalizado                          | Como usuario de Viora (Productor o Especialista), quiero gestionar mi acceso y perfil de manera segura y diferenciada, para disponer de una experiencia de bienvenida adaptada a mis necesidades profesionales y activar los beneficios del periodo de prueba sin fricciones.           |
| **EP02**    | Configuración de activos: gestión de parcelas y visibilidad profesional | Como usuario de Viora (Productor o Especialista), quiero configurar y gestionar mis activos principales (delimitación de parcelas agrícolas o publicación de mi portafolio profesional), para habilitar el monitoreo satelital, alimentar el motor predictivo con el historial agronómico necesario, y activar mi visibilidad en el ecosistema de conexión profesional.                        |
| **EP03**    | Consolidación de monitoreo agrometeorológico y predicción | Como Productor Olivarero, quiero que el sistema consolide y evalúe la información climática, satelital y predictiva, para conocer la situación real de mi parcela y tomar decisiones preventivas frente a anomalías ambientales. |
| **EP04** | Gestión de riesgos climáticos y mitigación por nutrición dinámica | Como Productor Olivarero, quiero recibir alertas de riesgo fenológico y prescripciones automáticas de nutrición dinámica, registrando su ejecución técnica y financiera, para mitigar los efectos de anomalías climáticas (como el Fenómeno ENOS), prevenir la vecería y asegurar la rentabilidad de mi unidad productiva. |
| **EP05**    | Vigilancia epidemiológica y prospección territorial | Como usuario de Viora (Productor o Especialista), quiero reportar síntomas biológicos para que el sistema evalúe el riesgo automáticamente, para detonar alertas preventivas en la comunidad y facilitar la conexión con especialistas en zonas críticas.                                                       |
| **EP06**    | Matchmaking y gestión de intervenciones fitosanitarias | Como usuario de Viora (Productor o Especialista), quiero gestionar la vinculación entre alertas activas y la capacidad técnica disponible en la zona, para establecer una respuesta inmediata y coordinada ante emergencias en el cultivo.                                                                                            |
| **EP07**    | Evaluación técnica, cotización y revelación de contacto                 | Como usuario de Viora, quiero formalizar la prestación del servicio mediante la revisión técnica del fundo, el envío de una propuesta económica y la visualización de los datos de contacto directo, para iniciar la mitigación de la plaga.                                                                         |
| **EP08**    | Moderación automatizada y sistema de strikes                            | Como usuario de Viora, quiero contar con un sistema de reporte de inconductas que evalúe automáticamente el comportamiento en la plataforma, para que los usuarios conflictivos sean sancionados sistemáticamente mediante la acumulación de reportes.                                                    |
| **EP09**    | Diagnóstico fitosanitario en campo y prescripción técnica               | Como Especialista en Control de Plagas, quiero revisar el historial de la parcela y registrar los hallazgos de mi inspección física, para poder emitir una prescripción precisa en el momento climático óptimo.                                                                                                       |
| **EP10**    | Ejecución de la intervención y trazabilidad operativa                   | Como Productor Olivarero, quiero certificar la ejecución de las prescripciones técnicas y asentar los gastos asociados, para cerrar el ciclo de la alerta fitosanitaria manteniendo la trazabilidad agronómica y financiera de mi campaña. |                                                                              |
| **EP11**    | Cierre del servicio, retroalimentación y actualización de portafolio    | Como usuario del Ecosistema Viora, quiero evaluar el impacto del servicio recibido indicando mi disposición a recontratar al técnico, para cerrar formalmente la asesoría y alimentar el algoritmo de recomendación del Ecosistema. |                                                                                      |
| **EP12**    | Ajustes de seguridad y programa de afiliados | Como usuario del Ecosistema Viora, quiero gestionar la seguridad de mi cuenta y participar en el programa de referidos, para proteger mis accesos y obtener beneficios económicos por invitar a nuevos actores a la red. |                                                                                  |
| **EP13**    | Gestión de suscripciones, pagos y facturación                           | Como usuario de Viora, quiero gestionar mi plan de suscripción y métodos de pago de forma segura, para mantener mi acceso a la plataforma o solicitar reembolsos según las políticas de garantía.                                                                                                                     |
| **EP14** | Descubrimiento y captación de interés        | Como Visitante, quiero comprender rápidamente el propósito de Viora y su solución a la incertidumbre climática, para determinar si el ecosistema es valioso para mí y decidir iniciar mi registro. |
|**EP15**| Segmentación de valor y oferta de servicios  | Como Visitante segmentado, quiero descubrir las herramientas específicas para mi rol y las condiciones de acceso, para confirmar que la plataforma cubre mis necesidades operativas sin riesgo financiero inicial. |
|**EP16**| Prueba social y respaldo institucional | Como Visitante cauteloso, quiero validar la credibilidad de la plataforma mediante experiencias de terceros, conocer al equipo y tener vías de comunicación, para sentir la confianza institucional necesaria antes de iniciar mi registro formal.  |                                                                           |
| **EP17** | API de Seguridad y Gestión de Identidad (IAM) | Como Developer, quiero implementar un sistema de Gestión de Identidad y acceso basado en tokens JWT, para garantizar la autenticación segura de los usuarios y la segregación de funciones según su rol jerárquico.|
| **EP18** | API de monitoreo agrometeorológico y predicción | Como Developer, quiero desarrollar el motor de integración con servicios satelitales y meteorológicos, para automatizar el registro de parcelas, el cálculo de vigor (NDVI), porciones de frío y la ingesta del historial agronómico. |
| **EP19** | API del marketplace y negociación del marketplace| Como Developer, quiero exponer los servicios del motor transaccional de búsqueda y contratación, para gestionar la disponibilidad de los especialistas y permitir el emparejamiento geolocalizado mediante cotizaciones digitales. |
| **EP20** | API de trazabilidad de campo y gobernanza comunitaria | Como Developer, quiero construir el sistema de monitoreo de intervenciones y moderación autónoma, para procesar reportes fitosanitarios, auditar recetas agrícolas, registrar gastos y aplicar sanciones automáticas ante inconductas. |
| **EP21** | API de detección y alertas epidemiológicas | Como Developer, quiero implementar el motor de prospección zonal y evaluación de sintomatología, para cruzar los reportes manuales con métricas NDVI y emitir alertas comunitarias sobre posibles plagas. |
| **EP22** | API de suscripciones y growth loop | Como Developer, quiero integrar el procesamiento de pagos asíncronos y el motor de recompensas por referidos, para automatizar el ciclo de vida de los planes (Trial/Premium) y fidelizar a los usuarios mediante incentivos financieros. |
